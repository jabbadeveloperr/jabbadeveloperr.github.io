# Звернення щодо законопроєкту №14394

Сайт для автоматизації листа народним депутатам України щодо проєкту Цивільного кодексу (№14394). Користувач заповнює свої дані та персональний коментар, переглядає лист і відправляє його (через поштовий клієнт або Gmail).

[Шаблон листа (Google Docs)](https://docs.google.com/document/d/1xxc4l5KCJseRNO_dNVzFXtJPs5Ax6hcPd4IE1TwfeBw)

## Можливості

- Заповнення полів, позначених у шаблоні (ПІБ, адреса, електронна пошта, персональний коментар).
- Список електронних пошт депутатів за замовчуванням — можна змінити перед відправкою.
- Кнопка **«Як виглядатиме лист»** — попередній перегляд тексту листа.
- Кнопка **«Відкрити у поштовому клієнті»** — відкриває стандартний поштовий клієнт (Outlook, Thunderbird, Gmail в браузері тощо) з заповненими полями «Кому», «Тема» та тілом листа.
- Опціонально: **авторизація через Gmail** і кнопка **«Відправити депутатам (Gmail)»** — відправка листа безпосередньо через Gmail API (потрібна налаштування OAuth у Google Cloud).

## GitHub Pages

1. Збережіть проєкт у репозиторії на GitHub.
2. У репозиторії: **Settings → Pages**.
3. **Source**: Deploy from a branch.
4. **Branch**: `main` (або `master`), папка **/ (root)**.
5. Збережіть — через хвилину сайт буде доступний за адресою `https://<username>.github.io/<repo>/`.

Без додаткової налаштування працюють кнопки «Як виглядатиме лист» та «Відкрити у поштовому клієнті».

## Відправка через Gmail (опціонально)

Щоб з’явилася кнопка «Відправити депутатам (Gmail)» і вхід через Google:

1. Перейдіть у [Google Cloud Console](https://console.cloud.google.com/).
2. Створіть проєкт або оберіть існуючий.
3. Увімкніть **Gmail API**: APIs & Services → Library → знайдіть «Gmail API» → Enable.
4. Створіть **OAuth 2.0 Client ID**: APIs & Services → Credentials → Create Credentials → OAuth client ID.  
   Тип: **Web application**.  
   Authorized JavaScript origins: додайте `https://<username>.github.io` та (для перевірки локально) `http://localhost:8000` або `http://127.0.0.1:5500`.  
   Authorized redirect URIs для статичного клієнта можна не заповнювати (використовується One Tap / token client).
5. Скопіюйте **Client ID** і вставте його в `config.js` у змінну `GOOGLE_CLIENT_ID`:

```js
window.GOOGLE_CLIENT_ID = 'ваш-client-id.apps.googleusercontent.com';
```

6. Зберіть зміни та задеплойте сайт на GitHub Pages. У списку тестових користувачів OAuth (якщо проєкт у режимі Testing) додайте свій Gmail.

**Важливо:** не публікуйте секрет OAuth (Client Secret) у публічному репозиторії. У статичному клієнті використовується лише Client ID; це допустимо для публічних односторінкових застосунків.

## Локальний запуск

Відкрийте `index.html` у браузері або використайте локальний сервер, наприклад:

```bash
npx serve .
# або
python -m http.server 8000
```

Потім відкрийте в браузері відповідну адресу (наприклад, `http://localhost:8000`).

## Файли

- `index.html` — форма, модальне вікно перегляду.
- `styles.css` — стилі.
- `app.js` — логіка форми, шаблон листа, перегляд, mailto, Gmail API.
- `config.js` — список пошт депутатів за замовчуванням та (опціонально) `GOOGLE_CLIENT_ID`.

## Ліцензія
